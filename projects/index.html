pyramid_appengine

This is a scaffold that creates a buildout based directory structure that is a bit easier to handle than virtualenv.

Technologies used

  • python
  • pyramid
  • buildout
  • google appengine

The Source

You can check out the source at github

I wrote this as part of the pycon/pyramid sprints in 2012. I had never written a scaffold before so that was fun. it was nice to take a step back from the environment we built up at batterii over the years and boil it down to the essentials that anyone can use.

This scaffold gets moderate usage from the community and one of the interesting side-effects of releasing this is that bugs filed are more about education on how appengine projects work and how python packaging works than about things not working in the scaffold itself. Though since it has to keep up with the app engine SDK release schedule, bugs do happen.

diredly

This is an experimental static site generator I'm maintaining and using for various projects.

Technologies used

  • python
  • pyramid

The Source

You can check out the source at github

Yes there are plenty of static site generators out there. Some are arguably better than this for a specific purpose. I've found that for me, sometimes I just need a little bit of a template, and other times I need to create content with a bit more structure more like a cms or blog.

So, I decided to see how far I could get abusing pyramids traversal mechanism and this is the result. I predict that in the future I'll either realize why I should use something else like pelican or sphinx, or I will have built up everything I typically need to build a staticish website.

webobtoolkit

This is a collection of utilities and wsgi middleware for building http clients using webob.

Technologies Used

  • python
  • webob

The Source

You can check out the source at github

Webob does a great job of handling all the messy intricacies of HTTP, being on appengine we use it a lot. A while back webob grew some rudimentary client functionality but it wasn't as robust as it could be. So as an experiment, I wrote some unit tests that were inspired by the requests quick start. Things like cookie handling between requests and handling gzipped responses was added as wsgi middleware allowing us to construct a nice http client that could be used on appengine and as a bonus the responses were the same type as what is used in our code already.
